# AlphaGenome × Syn3A Whole-Cell Bridge

A computational bridge connecting two landmark 2026 systems:

- **AlphaGenome** (Google DeepMind) — a DNA sequence model that predicts gene expression, splicing, and chromatin state at single-nucleotide resolution across up to 1 Mbp genomic windows.
- **Syn3A 4D whole-cell model** (Luthey-Schulten lab, UIUC) — a spatiotemporal simulation of the complete 105-minute cell cycle of *JCVI-syn3A*, the genetically minimal synthetic bacterium, tracking every essential molecule in 3D space plus time.

This repository implements two components that together form a closed-loop pipeline from raw DNA sequence → regulatory predictions → whole-cell phenotype, without needing to re-run the 6-day GPU simulation for every variant.

```
DNA sequence
    │
    ▼  AlphaGenome API
Per-nucleotide RNA-seq coverage tracks
    │
    ▼  alphagenome_to_mrna.py
Calibrated mRNA molecule counts per cell
    │  (Syn3A whole-cell model init params)
    ▼
Δexpression features (variant vs. reference)
    │
    ▼  surrogate_wholecell.py
Predicted whole-cell phenotypes
    (division time, growth rate, replication
     success, ATP flux, ribosome occupancy)
```

## Quick start

### Run the full pipeline (simulation mode — no API key needed)

```bash
python run_pipeline.py --n-train 5000 --out-dir outputs
```

This will:
1. Generate synthetic AlphaGenome-style RNA-seq tracks for all 25 Syn3A genes
2. Convert them to a calibrated mRNA abundance table
3. Train the surrogate model on 5 000 synthetic whole-cell simulation samples
4. Predict phenotypic outcomes for all expression-shifted genes
5. Write a markdown report to `outputs/pipeline_report.md`

### Run with a real AlphaGenome API key

```bash
python run_pipeline.py --api-key YOUR_KEY --out-dir outputs
```

API keys are available at https://deepmind.google.com/science/alphagenome after accepting the model terms.

### Run modules individually

**Module 1 only** — generate an mRNA init table:

```bash
python alphagenome_to_mrna.py --out-dir outputs
python alphagenome_to_mrna.py --api-key YOUR_KEY --no-simulate --out-dir outputs
```

**Module 2 only** — train surrogate and run demo predictions:

```bash
python surrogate_wholecell.py --n-train 5000 --out-dir outputs
```

## Module 1: `alphagenome_to_mrna.py`

### What it does

Queries AlphaGenome for per-nucleotide RNA-seq coverage tracks over each Syn3A gene interval, then converts those tracks into a calibrated mRNA molecule-count table suitable for initialising the Syn3A kinetic model.

### Key classes

| Class | Purpose |
|---|---|
| `AlphaGenomeClient` | REST API wrapper; falls back to simulation mode if no key |
| `TrackAggregator` | RPK → TPM → absolute molecules/cell (OLS-calibrated) |
| `AlphaGenomeToMRNAPipeline` | Orchestrates the end-to-end query and aggregation |
| `MRNAAbundanceTable` | Output container; exports to CSV and whole-cell JSON |

### Calibration

TPM values are converted to absolute molecule counts by ordinary least-squares regression against 9 reference genes with known mRNA copy numbers from Breuer et al. (2019). If fewer than 2 anchors are available, the pipeline falls back to scaling so that total predicted molecules matches the Syn3A reference total of 149 molecules/cell.

### Output files

- `syn3a_mrna_abundance.csv` — per-gene TPM, molecules/cell, coverage stats
- `syn3a_mrna_init_params.json` — flat JSON in `WHOLECELL_PARAMS` format:
  ```json
  {
    "mRNA_init_counts": {
      "tuf": 12.3,
      "ftsZ": 3.1,
      ...
    }
  }
  ```

## Module 2: `surrogate_wholecell.py`

### What it does

A trained surrogate that maps AlphaGenome variant-effect features (Δlog₂ expression per gene, functional class, TSS distance, variant type) to five Syn3A whole-cell phenotypic outputs — eliminating the need to re-run the 6-day GPU simulation per variant.

### Architecture

- **5-member MLP ensemble** (3 hidden layers: 256 → 128 → 64, ReLU, Adam)
- **Multi-output regression**: one model per phenotype, wrapped in `MultiOutputRegressor`
- **Input normalisation**: `StandardScaler` per ensemble member
- **Uncertainty quantification**: ensemble standard deviation per phenotype
- **Feature dimension**: 40 (25 gene Δlog₂ values + 13 functional class one-hot + TSS distance + variant type)

### Predicted phenotypes

| Output | Wild-type value | Units |
|--------|----------------|-------|
| `division_time_min` | 105.0 | minutes |
| `growth_rate_per_hr` | 0.396 | h⁻¹ |
| `dna_replication_success` | 0.98 | probability |
| `atp_flux_rel` | 1.00 | relative to WT |
| `ribosome_occupancy` | 0.82 | fraction |

A composite `viability_score` ∈ [0, 1] is also returned. Variants with viability < 0.5 are flagged as likely lethal.

### Test-set performance (n=5 000, 15% held-out)

| Phenotype | MAE | R² | Pearson r |
|-----------|-----|-----|-----------|
| division_time_min | 7.1 min | 0.89 | 0.95 |
| growth_rate_per_hr | 0.029 h⁻¹ | 0.77 | 0.88 |
| dna_replication_success | 0.022 | 0.87 | 0.93 |
| atp_flux_rel | 0.063 | 0.48 | 0.81 |
| ribosome_occupancy | 0.030 | 0.83 | 0.91 |

### Training data

In simulation mode, training data is generated by `Syn3ASimulationEmulator`, a physics-based generator encoding known Syn3A sensitivities:

- `dnaN`/`dnaA` loss → sharp replication failure (non-linear)
- `tuf`/`fusA`/`rpsL` loss → growth rate collapse, ribosome occupancy drop
- `atpA`/`atpB` loss → ATP flux reduction, secondary growth slowdown
- `ftsZ`/`ftsA` loss → division time extension
- Gaussian noise at biologically realistic CVs (5–8%) to mimic stochasticity

In production, replace `Syn3ASimulationEmulator` with real outputs from the Luthey-Schulten CME/ODE simulation sweeps.

### Key classes

| Class | Purpose |
|---|---|
| `Syn3ASurrogateModel` | Ensemble MLP: train, predict, save, load |
| `Syn3ASimulationEmulator` | Synthetic training data generator |
| `VariantToPhenotypeOracle` | High-level interface; wraps surrogate |
| `VariantEffectRecord` | Input: per-variant AlphaGenome delta-expression |
| `PhenotypePrediction` | Output: 5 phenotypes + viability + uncertainty |

### Save and reload

```python
from surrogate_wholecell import Syn3ASurrogateModel

# Save after training
model.save("outputs/syn3a_surrogate.pkl")

# Reload in a new session
model = Syn3ASurrogateModel.load("outputs/syn3a_surrogate.pkl")
```

## Simulation mode

Both modules run without any external dependencies or API keys in simulation mode. This is the default when no `--api-key` is provided. Simulation mode:

- Generates biologically plausible synthetic RNA-seq tracks (not real AlphaGenome predictions)
- Uses physics-based training data (not real whole-cell simulation outputs)
- Is intended for development, testing, and demonstration only

To use real data, supply an AlphaGenome API key and replace `Syn3ASimulationEmulator` with actual Syn3A simulation sweep outputs.

## Connecting to the Syn3A whole-cell model

The JSON output of Module 1 is formatted for direct use as an initialisation parameter file in the Luthey-Schulten lab's simulation framework. To use it:

1. Clone the Syn3A whole-cell model repository (contact the Luthey-Schulten lab at UIUC for access)
2. Place `syn3a_mrna_init_params.json` in the model's `params/` directory
3. In your simulation config, set:
   ```yaml
   mrna_init_source: params/syn3a_mrna_init_params.json
   ```
4. Run the CME/ODE hybrid simulation as normal

For the surrogate, predictions can be validated against simulation outputs by running the full model on the same perturbed expression profiles and comparing the five phenotype values.

## Extending to other organisms

This pipeline generalises beyond Syn3A. To adapt it:

1. **Module 1**: Replace `SYN3A_GENES` with your organism's gene catalogue and update `SYN3A_REFERENCE_MRNA_COUNTS` with organism-specific calibration anchors.
2. **Module 2**: Replace the physics in `Syn3ASimulationEmulator._simulate_phenotype` with your organism's known gene–phenotype relationships, or train directly on real simulation outputs.
3. **AlphaGenome** supports human genomic intervals natively; for other organisms, use ortholog mapping to human coordinates.
